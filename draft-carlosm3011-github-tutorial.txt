1.  Introduction

   Useful ideas for using GitHub for document production.

2.  Conventions and Definitions

   The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT",
   "SHOULD", "SHOULD NOT", "RECOMMENDED", "NOT RECOMMENDED", "MAY", and
   "OPTIONAL" in this document are to be interpreted as described in
   BCP 14 [RFC2119] [RFC8174] when, and only when, they appear in all
   capitals, as shown here.

3.  Bootstrapping

3.1.  Steps:

   *  Clone the template named "internet-draft-template" by "Martin J
      Thompson".

   *  Edit the name of the md file to have a proper draft name.

   *  Fill in as much metadata as possible.

   *  Commit

   The URL for the github template is: https://github.com/martinthomson/
   internet-draft-template

3.2.  What will happen next ?

   A set of Github actions will be triggered which will reshape the
   repository, include boilerplate in the document, create a README and
   a LICENSE file.

4.  Normal Editing

   Edits can be made in-line on Github web or using the GH desktop app.
   Changes can be either committed or used to create a pull request.

   PRs are RECOMMENDED.

5.  Sending to the datatracker

   This is done creating a TAG with the exact name of the draft.  This
   will run the submission process.
